# AIWF

AIWF is a Python CLI for a task-centered development loop for AI-assisted software work.

The current implementation is the v2 refactor in progress. What is implemented today is:

- `.ai/` workspace initialization
- task lifecycle records under `.ai/tasks/`
- gate execution and verification evidence under `.ai/runs/`
- task state transitions: `defined -> in_progress -> verifying -> done/failed/blocked`
- JSON schema validation for persisted records
- telemetry event logging

What is **not** implemented today:

- project map / feature map
- project onboarding scan
- git automation
- multi-agent orchestration
- old `develop` / `roles` / `dispatch` workflow from earlier docs

## Current CLI Surface

The repository's current CLI entrypoint is [src/aiwf/cli/main.py](/home/gloria/tianyue/aiwf-framework-scaffold/src/aiwf/cli/main.py).

Implemented commands:

- `aiwf init` -> prints JSON with `workspace`, `config`, and `state` paths
- `aiwf status`
- `aiwf verify`
- `aiwf task new "<title>" [--scope "..."] [--accept "..."] [--files "a.py,b.py"]`
- `aiwf task start [task-id]`
- `aiwf task current`
- `aiwf task list [--status ...]`
- `aiwf task verify [task-id]`
- `aiwf task close [task-id]`
- `aiwf task block [task-id] --reason "..."`
- `aiwf task unblock <task-id>`
- `aiwf task retry [task-id]`

## Repository Layout

- `src/aiwf/`: project code
- `schemas/`: JSON contracts for persisted AIWF data
- `tests/`: test suite for CLI, engine, storage, and schemas
- `docs/`: development guidance and current project records
- `.ai/`: runtime workspace data and evidence generated by AIWF

Important: `.ai/` in this repository contains runtime artifacts and some older local leftovers. Do not treat it as the design source of truth for the repository itself.
